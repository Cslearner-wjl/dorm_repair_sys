# 宿舍报修管理系统前端页面设计说明

## 设计定位

这是一个校园后勤多角色业务系统，核心目标是让学生快速提交与追踪报修，让管理员高效审核派单，让维修人员按状态推进任务。界面参考深青色侧栏与白色内容区的校园管理台风格，后续可直接迁移到 Vue 3、Vite、Element Plus。

设计参数：

- 视觉变化：3，保证页面克制、稳定、适合课程验收。
- 动效强度：2，仅保留按钮反馈、提示消息等必要状态。
- 信息密度：8，列表、筛选、统计卡片、进度步骤和表单字段优先，减少装饰。

## 页面清单

| 页面 | 角色 | 设计重点 |
| --- | --- | --- |
| 登录页 | 全部 | 一个入口登录，登录后根据角色进入对应首页。 |
| 注册页 | 学生、维修人员 | 学号或工号、姓名、手机号、密码、角色。 |
| 学生首页 | 学生 | 个人报修统计、最近报修、待确认提醒。 |
| 新增报修页 | 学生 | 必填校验清晰，图片字段先预留。 |
| 我的报修页 | 学生 | 按状态、类型、时间筛选，支持撤销和查看详情。 |
| 报修详情页 | 相关用户 | 展示基础信息、状态、处理记录、当前角色可操作按钮。 |
| 管理员首页 | 管理员 | 总数、待受理、处理中、已完成、分类统计、人员负载。 |
| 派单工作台 | 管理员 | 集中展示待受理、已受理、需重派工单，结合维修人员负载辅助派单。 |
| 报修管理页 | 管理员 | 审核、驳回、派单、异常状态处理。 |
| 用户管理页 | 管理员 | 用户角色、启用禁用、角色筛选。 |
| 维修任务页 | 维修人员 | 我的任务、开始处理、提交完成、申请重派。 |
| 维修处理提交页 | 维修人员 | 填写处理结果、维修用时、耗材记录，提交完成或申请重派。 |
| 学生确认评价页 | 学生 | 核对维修结果，提交评分和文字评价，关闭工单。 |

## 信息架构

- 左侧：系统标识、当前角色切换、角色菜单、流程说明，深青色背景，绿色激活项。
- 顶部：当前页面标题、日期、通知、头像、搜索框、导出和主要操作。
- 主区：统计卡片、筛选区、表格、详情时间线、表单。
- 状态：待受理、已受理、已派单、处理中、待确认、已完成、已驳回、已撤销、需重派。

## 组件规范

- 颜色：主色为绿色，用于主按钮、焦点态、关键进度；侧栏为深青色。
- 圆角：统一 6px 到 12px，按钮 6px，卡片 8px，登录页大容器 12px。
- 表格：适合几十到几百条数据，列包含工单号、问题、宿舍、类型、状态、维修人员、时间、操作。
- 表单：必填项就地提示，手机号和空值在前端先拦截，后端再次校验。
- 时间线：用于报修详情，解释每次状态变更和处理记录。
- 图片资产：登录页插画、品牌标识和报修缩略图已放入 `assets/`，原型阶段来自参考图裁切。

## Vue 与 Element Plus 落地建议

- `src/layouts/AppLayout.vue`：左侧菜单、顶部栏、权限菜单。
- `src/views/auth/Login.vue`、`Register.vue`：认证页面。
- `src/views/student/Dashboard.vue`、`RepairCreate.vue`、`MyRepairs.vue`。
- `src/views/student/RepairFeedback.vue`：学生确认维修结果与评价。
- `src/views/admin/Dashboard.vue`、`DispatchWorkbench.vue`、`RepairManage.vue`、`UserManage.vue`。
- `src/views/repair/TaskList.vue`、`TaskSubmit.vue`、`TaskDetail.vue`。
- `src/components/StatusTag.vue`：统一状态颜色和文案。
- `src/components/RepairTable.vue`：复用工单列表。
- `src/components/RepairTimeline.vue`：复用处理记录。
- `src/api/`：按 auth、repairs、statistics、users 拆分接口。

## 原型使用方式

打开 `frontend-prototype/index.html`，通过左侧“当前角色”切换学生、管理员、维修人员视角。该原型用于确认页面结构、状态文案和关键操作位置，不依赖后端接口。
