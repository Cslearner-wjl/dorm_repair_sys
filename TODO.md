# 宿舍报修管理系统 Agent 开发任务清单

## 0. 项目开发原则

本项目是《软件系统设计与开发实践》课程项目，目标是完成一个可运行、可演示、数据可持久化的小型 Web 业务系统。

开发时必须遵守以下原则：

* 先完成主流程闭环，再做优化和扩展。
* 所有核心数据必须写入 MySQL，禁止只在前端写死演示数据。
* 每个核心功能必须同时具备：页面、接口、数据库记录、测试结果。
* 状态流转必须由后端 Service 层统一校验，前端只负责根据状态展示可操作按钮。
* 所有业务接口必须做登录校验和角色权限校验。
* 不优先实现图片上传、消息通知、Excel 导出、移动端适配、超时提醒等扩展功能。
* 优先保证系统能完成课程验收演示：学生提交报修 -> 管理员审核派单 -> 维修人员处理 -> 学生确认评价 -> 管理员查看统计。
