# 《软件系统设计与开发实践》课程任务书

---

## 一、课程简介

《软件系统设计与开发实践》课程的主要目的是培养学生综合运用所学知识，分析和解决实际问题的能力。促使学生将理论方法与实际应用相结合，能够全面而辩证地分析和处理问题，熟练掌握软件开发的基本技术和工作过程，培养严谨认真的科学态度和务实的工作作风。

## 二、课程目标

课程核心目标如下：

1. 掌握完整软件工程项目开发流程，涵盖需求分析、系统设计、编码实现、测试优化、文档撰写与项目部署全环节，建立标准化工程开发思维。
2. 能够独立完成小型 Web 系统的架构选型、模块划分、数据库设计、接口设计与功能开发，具备基础的系统拆解与落地实现能力。
3. 提升团队协作、任务分工、进度管控、问题排查与沟通复盘能力，契合企业软件开发团队工作模式。

## 三、课程任务

### 1. 总体要求

各小组自主选题，完成一个功能完整、可正常运行、逻辑自洽的小型 Web 业务系统。系统主要包含：用户访问入口、基础业务功能、数据持久化存储、基础权限/角色区分、简单交互页面与异常处理等。项目不限制具体开发框架与技术栈，但需保证项目闭环可用。同时满足轻量化、实用性、完整性原则，符合软件工程模块化、规范化开发思想。以下为项目具体要求：

- 前后端交互正常，数据可正常新增、查询、更新、删除；
- 实现数据持久化存储，页面数据与数据库实时同步；
- 具备基础用户功能：注册/登录、权限区分（普通用户/管理员）；
- 具备基础异常处理：表单校验、错误提示、非法访问拦截；
- 代码结构模块化、分层清晰，无大量冗余无效代码，命名规范。
- 系统界面美观大方、简洁明了，交互逻辑清晰。

### 2. 分组要求

每组 3–5 人，自由组队，组内明确分工。每组需明确项目经理、前端开发、后端开发、测试、文档编写等角色，鼓励角色轮换、全员参与核心开发，确保每位学生均有实操产出。同时，需明确每位成员具体工作内容、工作量占比、贡献说明（作为个人评分依据）。

## 四、技术选型规范

为适配不同学生技术基础，本次实践不指定、不强制任何技术框架，学生可根据自身掌握程度自主选型，仅要求技术栈成熟稳定、可落地实现、适配 Web 系统开发。提供主流选型参考，供不同基础学生选择：

### 1. 前端技术（任选其一或自主搭配）

- **入门轻量化**：HTML+CSS+JavaScript、Bootstrap、LayUI、JQuery。
- **进阶主流**：Vue2/Vue3、React、Element Plus、Ant Design 等主流前端框架。

### 2. 后端技术（任选其一或自主搭配）

- **Java 方向**：SSM、Spring Boot、Spring MVC（入门友好，适配多数学生基础）。
- **Python 方向**：Django、Flask、FastAPI（轻量化、开发高效）。
- **其他方向**：Node.js、PHP、.NET 等成熟 Web 后端技术栈均可使用。

### 3. 数据库与工具

- **数据库**：MySQL、SQLite、PostgreSQL 等关系型数据库优先，禁止无数据库纯静态页面项目。
- **开发工具、版本控制、测试工具**不限，建议统一使用 Git 进行版本管理，培养工程规范。

## 五、推荐选题方向（可选、可自主创新）

学生可自主拟定选题，选题需贴合小型业务场景、工作量适中、难度适中，禁止超大场景、无法落地或纯静态展示项目。推荐选题参考：

- **校园类**：校园图书借阅系统、学生信息管理系统、校园失物招领系统、课程作业提交系统、社团管理系统。
- **生活工具类**：个人记账系统、简易票务预约系统、宿舍报修管理系统、在线留言反馈系统。
- **通用管理类**：小型商品进销存系统、员工信息管理系统、简易博客/资讯管理系统、考勤管理系统。

## 六、课程报告内容要求

课程报告是学习过程与成果的总结，集中反映了同学们的理论基础、实践水平及将二者有机结合的能力。要求按照软件工程的开发流程和规范撰写课程报告。主要包含需求分析、技术选型、系统架构、模块设计、数据库设计、接口说明、开发难点与解决方案等内容。

根据下面的内容要求，提交课程报告一份（各组员不得雷同）。

- 本课程报告应包含以下几个部分：封面、目录、正文、总结、参考文献、附录、评分表等。
- 报告总字数不少于 3000 中文字。
- 观点正确，论述深入全面，图文并茂，条理分明。

## 七、课程报告格式要求

标题应简短、明确、有概括性。标题字数要适当，一般不超过 25 个字。如果有细节可以分成主标题和副标题。使用 A4 打印纸打印。

| 层级 | 格式要求 |
|------|----------|
| 第一级（章）题序和题名 | 黑体小二号字 |
| 第二级（条）题序和题名 | 黑体小三号字 |
| 第三级（条）题序和题名 | 黑体四号字 |
| 第四级（条）题序和题名 | 黑体小四号字 |
| 正文各层次内容 | 宋体小四号字（英文用新罗马体12），固定值，24磅 |
