# 宿舍报修管理系统课程验收资料

本目录用于集中存放课程答辩和验收提交材料。

## 一、建议提交材料

| 序号 | 材料 | 文件或目录 | 是否建议提交 | 说明 |
| --- | --- | --- | --- | --- |
| 1 | 答辩 PPT | `答辩.pptx` | 必交 | 用于课堂答辩展示项目背景、功能、架构、测试和总结 |
| 2 | 课程报告 | `软件zy2301_吴家乐.docx` | 必交 | 按课程任务书要求提交 Word 版报告 |
| 3 | 课程报告 Markdown 版 | `课程报告.md` | 建议提交 | 便于老师或小组成员查看、修改和留档 |
| 4 | 项目源代码 | GitHub 仓库或本地生成的 `宿舍报修管理系统源码.zip` | 必交 | 包含 `server/`、`frontend/`、`sql/`、`docs/`、`README.md` |
| 5 | 数据库脚本 | `sql/init.sql`、`sql/demo-data.sql` | 必交 | 用于创建数据库结构和导入演示数据 |
| 6 | 项目说明 | `项目说明-README.md` | 建议提交 | 说明项目功能、技术栈和运行方式 |
| 7 | 部署说明 | `部署运行说明.md` | 建议提交 | 说明前后端、数据库和环境变量配置 |
| 8 | API 文档 | `API接口文档.md` | 建议提交 | 说明后端接口、请求参数和响应结构 |
| 9 | 需求文档 | `需求文档.md` | 建议提交 | 说明系统需求、角色和功能范围 |
| 10 | 测试记录 | `接口测试记录.md`、`screenshots/` | 建议提交 | 用于证明功能测试、构建验证和数据库验证结果 |
| 11 | TXT 文档版 | `txt/` | 可选 | 将主要 Markdown 项目文档整理为 `.txt` 格式，便于直接提交或查看 |

## 二、如果老师只要求精简提交

优先提交以下内容：

1. `答辩.pptx`
2. `软件zy2301_吴家乐.docx`
3. GitHub 仓库或本地生成的 `宿舍报修管理系统源码.zip`
4. `sql/init.sql` 和 `sql/demo-data.sql`
5. `README.md` 或本目录下的 `项目说明-README.md`

## 三、源码压缩建议

压缩项目源码时建议包含：

- `server/`
- `frontend/`
- `sql/`
- `docs/`
- `README.md`
- `start-all.ps1`

建议排除：

- `frontend/node_modules/`
- `frontend/dist/`
- `server/target/`
- `.git/`
- `.idea/`
- `.codex-run/`
- `server/uploads/` 中的临时上传文件

## 四、提交前注意

- 正式课程报告以 `软件zy2301_吴家乐.docx` 为准。
- `课程报告.md` 是 Markdown 备份，如需提交该版本，请先补充班级、姓名、学号和指导教师。
- `txt/` 是主要项目文档的 TXT 格式副本，内容来源于本目录的 Markdown 文档。
- `答辩.pptx` 使用现有答辩稿，可直接放映；如老师要求可编辑文字版 PPT，需要再基于现有内容单独制作。

## 五、答辩演示顺序

建议按以下顺序演示：

1. 登录页面和三类演示账号
2. 学生提交报修
3. 管理员审核、驳回或派单
4. 维修人员开始处理、提交完成或申请重派
5. 学生确认完成并评价
6. 管理员查看统计分析和用户管理
7. 展示数据库持久化、测试结果和项目文档
