# 答辩演示流程

## 1. 项目简介

项目名称：宿舍报修管理系统  
项目类型：前后端分离 Web 业务系统  
核心目标：解决宿舍报修登记分散、进度不透明、责任链条不清晰和统计困难等问题。

## 2. 演示账号

| 角色 | 账号 | 密码 | 演示内容 |
| --- | --- | --- | --- |
| 学生 | `student001` | `123456` | 提交报修、查看进度、确认完成、评价 |
| 管理员 | `admin001` | `123456` | 审核、派单、用户管理、统计分析 |
| 维修人员 | `repair001` | `123456` | 查看任务、开始处理、提交完成、申请重派 |

## 3. 推荐演示步骤

1. 打开前端首页，展示登录注册入口。
2. 使用学生账号登录，展示学生首页、我的报修和提交报修页面。
3. 新建一条报修单，说明表单校验和数据持久化。
4. 切换管理员账号，展示待受理工单，执行审核和派单。
5. 切换维修人员账号，展示个人任务，执行开始处理和提交完成。
6. 切回学生账号，确认完成并提交评价。
7. 切换管理员账号，展示统计首页、状态分布、分类分布和用户管理。
8. 展示数据库表数据、接口测试记录、后端单元测试和前端构建结果。

## 4. 重点讲解内容

- 三类角色权限：学生、管理员、维修人员权限边界清晰。
- 工单状态流转：待受理、已受理、已派单、处理中、待确认、已完成等状态闭环。
- 前后端分离：React 前端通过 Axios 调用 Spring Boot REST API。
- 数据持久化：MySQL 保存用户、工单、处理记录和评价。
- 安全控制：JWT 鉴权、角色校验、公开注册仅允许学生。
- 测试验证：接口测试、数据库验证、后端单元测试和前端构建验证。

## 5. 常见答辩问题准备

| 问题 | 回答要点 |
| --- | --- |
| 为什么选择这个题目？ | 贴近校园生活，业务流程清晰，适合展示 Web 系统开发完整闭环 |
| 为什么采用前后端分离？ | 前端负责交互体验，后端负责业务规则和数据，职责更清晰，便于维护 |
| 为什么用 React？ | 组件化能力强，适合多角色、多页面和多状态交互 |
| 为什么用 Spring Boot？ | 生态成熟，适合快速构建 REST API 和分层业务系统 |
| 如何防止越权？ | 后端使用 JWT 识别用户，通过角色注解和 Service 层校验控制权限 |
| 工单状态如何保证正确？ | 状态流转集中在后端 Service 层控制，非法状态操作会被拒绝 |
| 数据是否真实持久化？ | 用户、工单、处理记录、评价均写入 MySQL，可通过数据库截图和 SQL 验证 |
| 项目测试做了什么？ | 覆盖权限、参数校验、状态流转、完整业务流程、单元测试和构建验证 |
